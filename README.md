# LeBron · Data

A data visualization site for LeBron James, built locally on top of Kaggle CSVs.

## Views

1. **Connections** (`/`) — every player LeBron has shared a court with, clustered by team. (Built.)
2. **Shots** (`/shots`) — animated career-long shot chart on a basketball court, filterable by opposing team and period. *(Coming.)*
3. **Minutes** (`/minutes`) — chronological grid of every game played, colored by minutes on the floor. *(Coming.)*

## Stack

- Next.js 16 (App Router) · TypeScript · Tailwind v4 · shadcn/ui
- `better-sqlite3` for the local DB (file at `data/nba.db`)
- `react-force-graph-2d` for the connections view

## Data

The app expects player-game level data dumped to `data/raw/` and seeded into `data/nba.db`.

### Recommended Kaggle source

Look for a Kaggle dataset along the lines of **"NBA, ABA, BAA Stats"** (look for a CSV named `PlayerStatistics.csv` with one row per player per game). Drop the file at:

```
data/raw/PlayerStatistics.csv
```

The seeder is forgiving — it accepts any of these column-name variants (case-insensitive):

| Concept           | Variants                                                                 |
| ----------------- | ------------------------------------------------------------------------ |
| Player ID         | `personId`, `playerId`                                                   |
| Player name       | `firstName`/`lastName`, or `playerName`/`fullName`                       |
| Game ID           | `gameId`                                                                 |
| Date              | `gameDate`, `date`                                                       |
| Player's team     | `playerteamCity`+`playerteamName`, or `playerTeam`                       |
| Opponent's team   | `opponentteamCity`+`opponentteamName`, or `opponentTeam`                 |
| Minutes / Points  | `numMinutes` / `points`                                                  |

### Seed

```bash
npm install
npm run seed     # reads data/raw/*.csv, writes data/nba.db
npm run dev      # http://localhost:3000
```

The seed log will tell you how many appearances were inserted, whether LeBron was found, and how many distinct players he has faced.
